# Context Switch(문맥전환)
### 문맥전환이란?
**문맥 전환**(Context Switching)은 CPU가 한 프로세스(또는 스레드)에서 다른 프로세스로 전환될 때 일어나는 과정이다.  

운영체제는 동시에 여러 프로그램을 실행하는 것처럼 보이게 하기 위해
실제로는 CPU를 여러 프로세스가 번갈아 가며 사용하게 만든다.
이때 “지금 실행 중인 프로세스의 상태를 잠시 저장하고,
다른 프로세스의 상태를 불러오는” 작업이 필요하다.

#### 여기서 문맥이란?
**문맥**(Context)은 앞서 나왔듯 CPU가 어떤 프로세스를 실행하고 있는지에 대한 모든 정보이며, PCB 안에 저장된다.
|구분|	예시|
|---|---|
레지스터 값|	PC(Program Counter), Stack Pointer 등
프로세스 상태|	Ready, Running, Waiting 등
메모리 맵 정보|	코드/데이터/스택의 위치 등
파일 핸들, 권한 정보	|해당 프로세스가 접근 중인 자원

### 문맥전환이 필요한 이유
- 여러 프로세스와 스레드들을 동시에 실행시키기 위해(그렇게 보이기 위해)
- 여러 프로세스와 스레드들이 공정하게 CPU 시간을 나눠 갖기 위해
- 높은 우선순위의 작업이 빠르게 처리될 수 있게하기 위해
